How to Create a VPC Using Terraform

Terraform is an Infrastructure as Code (IaC) tool that allows you to automate cloud resource management. One of the common tasks that Terraform helps with is setting up a Virtual Private Cloud (VPC) on AWS. This blog will guide you through creating a VPC and its associated Internet Gateway.

Understanding VPC and Its Importance

A Virtual Private Cloud (VPC) is a virtual network within your AWS account. It allows you to securely launch and manage resources like EC2 instances, databases, and more. With VPC, you can control your network environment by setting up subnets, route tables, and network gateways.

Creating a VPC using Terraform ensures consistency, repeatability, and version control in your infrastructure setup. By writing simple configuration files, you can deploy a fully functional VPC in a matter of minutes.

Creating a VPC with Terraform

To create a VPC in AWS using Terraform, you need to define two main resources:

  • aws_vpc: This resource creates a virtual private cloud with a specified CIDR block.
  • aws_internet_gateway: This resource attaches an Internet Gateway to your VPC, allowing access to the internet.

Example of Terraform Configuration for VPC Creation:

In your Terraform configuration, you would define a resource for the VPC, specifying the CIDR block you want to use. For example, you can use a CIDR block like `10.20.0.0/16` for your VPC.

Additionally, you would define an Internet Gateway (IGW) resource, linking it to your VPC to enable internet connectivity for instances within that VPC.
